学习爬虫逆向的晋升之路和心态历程

习爬虫逆向的晋升之路和心态历程

自我介绍

我是专科毕业，大学主要就是铁路。开火车专业的，铁道机车。我会开火车。铁路里面绝大部分都是关系户，我关系不硬，没进去。

我毕业你后第一份工作是，亲戚在北京，就去了北京做清洗油烟机的活，去给亲戚干了一个月，都是晚上干活，一晚上做2-3单，就天亮了。

一天 150。我干了一个月，熬夜实在顶不住了，给我脸都熬的发黄，清洗的东西用的还是生石碱，石碱水碰到皮肤，烧疼。

然后在一家小公司做快递打包。干了一个多月，这家公司被举报说，零食是假货，老板就连夜跑路了。又去圆通快递，工资给我开的 3000，做了半年，老板涨了 500，全年无休，一干就是一年半。

在干快递的时候，我感觉日复一日，每天都重复，所以想学点啥，于是就在有空的时候，用老板的电脑学学 Python，那时经常看见 Python 的小视频广告。

当时 Python 语法啥的，我都不知道，我就硬着头皮敲，基本上都是照抄，抄多了就会了，导致我现在的学习习惯就是，先做，遇到不会了再补。

我其实一直都是属于白嫖党，因为家里也是一直很省吃俭用，我也不会把钱画在么必要的上面，因为白嫖，所以我的第一个爬虫目标就是抓网易云的音乐，然后学了两个月，就做出来一个音乐播放器。

干到同年10月。公司太黑奴了，他们老是有那种紧急任务，我第一份工作，又比较老实，干过好几次通宵，第一个工作还是单休，节假日休息也打折。国庆，不让我连歇5天，我就跟主管急眼了，就有了不干的想法。

在元章的帮助下，我搞出来了 XX 滑块，拿着 XX 滑块投简历，确实工作好找了，基本上投一个中一个。然后就跳槽到了郑州的一个公司，薪资待遇啥的都上来了，老板也是比较赏识我，我争取到了大小周，薪资9k。郑州这个公司，因为做 tk 的，然后还要做商详，我就开始搞 APP。

学习能力

郑哥：我到现在也是基本上没有讨论技术的人，我圈子也很小，活跃的群就郑州猿人学群，现在就跟黎明哥聊技术聊的比较多，他也算是我认识的技术中最好的了。其他基本上碰见问题就自己解决，也习惯自己解决了，遇见挫折、难题，解决不掉的问题，都不会影响整个学习兴趣。

技术等级

郑哥：感觉，算法这块没啥大问题了，现在在研究改机方向了。

其实现在搞某XX短视频也是头疼的很。之前做XX滑块是最痛苦的，那时候jsvmp 都不知道，硬着头皮，各种找资料，还有自己的猜想，搞出来了，感觉到现在，都没有遇到像那时候滑块时，手足无措的情况了。

rpc -> xposed -> unidbg -> 纯算。都是为了解决实际业务而学。就想各种办法去解决，去百度，去问 AI，去了解解决问题需要什么技术点，然后去学习技术点，再回过来解决。

我遇到问题就不断尝试，去猜，去百度搜索碎片化的信息，只要含有滑块的帖子，微信聊天记录，我都会记下来，然后拿去实战看看。

那段时间，做梦都在抠滑块，那时候知识储备量太少了，做XX滑块难度有点太大了。

学习建议

我感觉，不要一直刷视频，一直看帖子，多上手操作，不要因为 APP 是头部大厂，代码看不懂，就打退堂鼓，可以一点一点做，一天解决一点，大厂算法也只是时间问题。我做什么目的性都比较强，APP 逆向的知识点我也没学完，都是遇到了，再去补，有些不重要的就跳过去了，主打一个效率。

先找他的加密，然后确定加密在 java 还是 so ，在 so 就会先尝试用 unidbg 跑跑看。

unidbg 跑不起来，用我自己写的工具再试试，如果有业务需求的话，会优先用 xposed 插件解决。

爬虫逆向趋势

按照我现在的知识和了解，以后可能都是在改机方向的，现在能搞定算法的人太多了,每个公司都不缺能搞定算法的大佬，而且算法费时费力，用纯协议去跑业务，会遇到行为轨迹的风控。

面试的几个大厂和小厂，他们的业务场景好几个都是改机，现在 AI 越来越厉害，除非可以用纯协议补出完整的行为轨迹，但是这个太费事费力，算法稍微改动一下，整个协议都要修改。

有两个大厂就不说名字了，他们的业务是抓大厂的商品数据，两个都是改机做，他们不缺能做算法的人，聊的最多的还是在风控这块，然后他们给我说他们正在跑的项目用的就是改机+截图ocr，相对于协议还是稳定的，检测再厉害，不可能突破底层吧。

感觉以后内核级别的会越来越多，但是内核又比较麻烦，有个大厂给我提了一下，说改 aosp 那种源码没有意义，因为国内没有 aosp 的市场。

薪资待遇

我们做逆向都是半路出家，成长路径绝大部分都是：先使用工具实操先上手，之后再反补理论，等实操有经验后，会清楚的知道自己缺哪一块的理论知识，再针对性的学习理论，进步就很快。

每个人都有自己不同的生活轨迹，希望每个人都能在自己的生活成长路径中找到最优解，共勉。
